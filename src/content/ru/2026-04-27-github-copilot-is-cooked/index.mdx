Простите за кликбейтный заголовок. Просто мне кажется, после этих изменений GitHub проиграет Claude.

2 часа назад получил [письмо от GitHub](https://github.blog/news-insights/company-news/github-copilot-is-moving-to-usage-based-billing), что с 1 июня Copilot уходит от модели с реквестами к модели с токенами. То есть к такой же модели, которая используется у Claude.

Реквест — это взаимодействие пользователя с LLM. Например, отправка промпта в чате в VS Code — это один реквест. В Pro подписке даётся 300 реквестов на месяц. Мне этого всегда хватало, что дома при работе с пет-проектами, что на работе.

При достаточном умении, можно написать промт, который позволит за 1 реквест реализовать крупную фичу. Просто подробно описываешь желаемые изменения, указываешь тулы, просишь агента быть автономным. Агент начнёт работать, писать код, сжигать токены. И всё это за 1 реквест! 

Это было основное преимущество GitHub перед Claude. Но, видимо, таких умельцев стало слишком много, что для GitHub это стало невыгодным.

Кажется, нанять мидла скоро станет дешевле, чем подписка на ИИ...
